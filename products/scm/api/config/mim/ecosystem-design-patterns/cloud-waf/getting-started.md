If you've made it here, you should already have a general understanding about [Cloud WAFs and how they use machine identities](/scm/api/config/mim/ecosystem-design-patterns/cloud-waf) and you're ready to get started developing your **Cloud WAF** solution.

## High-level Process

1. Create a [Certificate Manager - SaaS](https://vaas.venafi.com) account if you don't have one already.
1. Decide on a programming language and understand what tooling, if any, your solution will use (If you aren't sure yet, [this](https://venafi-dev-onboarding.paperform.co/) might help 🙂).
1. Start building!
1. Perform [functional testing](/scm/api/config/mim/ecosystem-design-patterns/cloud-waf/ecosystem-design-patterns-cloud-waf-testing).
1. Get [certified](/scm/api/config/mim/certification)!

## Getting Help

If you aren't able to find what you're looking for, or have a specific question related to your use case, please [post a question](https://community.venafi.com/ask-the-community-23) to the Developer Forum section of Venafi's Warrior Community or send us an [email](mailto:ecosystem@venafi.com?subject=Developer+Question-Cloud+WAF).
