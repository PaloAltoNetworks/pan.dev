## Introduction

The most consistent and standard method for deploying *Workload Identity Manager* (formerly known as *Firefly*) in [Kubernetes](https://kubernetes.io/) is to use [Helm](https://helm.sh/) and Venafi provides [official Helm charts](https://gallery.ecr.aws/venafi-images/helm/firefly) to streamline these deployments.  Click [here](https://docs.venafi.cloud/firefly/deploy/kubernetes/) for more information about deploying *Workload Identity Manager* in Kubernetes.

## Chart Values

Helm allows customization through variables whose values are defined by a [values file](https://helm.sh/docs/chart_template_guide/values_files/).  The table below lists the values available for configuring the official *Workload Identity Manager* Helm charts using a `values.yaml` file.  Please note that some of the values are required.

> ✅ Tip
>
> You can execute the following command to generate a starter Helm values file (with comments) for deploying *Workload Identity Manager* in a Kubernetes cluster:
> ```bash
> helm show values oci://registry.venafi.cloud/public/venafi-images/helm/firefly --version v1.6.0 > firefly-values.yaml
> ```
